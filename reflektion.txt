Reflektion inloggningssida

Jag började uppgiften med att skapa issue för att göra en sida med html element och css,
så att sidan skulle vara färdig och jag kunde börja med koden.
Jag gjorde ett issue för inlogg och att spara användaren.
Jag visste inte rikgit hur jag skulle gå till väga med att leda användare till en ny sida,
men jag gjorde till slut en funktion som skapade de nya html-elementen för användaren när
man loggar in. 

Det smidigaste sättet jag kom på för att skapa ny sida åt användaren, var att skapa nya element
via en funktion, och lägga till dom i en redan existerande div tag, wrapper.
Därefter lägga till en ny eventlistener för den nya knappen, som kallar på en
ny funktion som tar bort alla element i div tag wrapper. Jag hittade metoden för att ta
bort alla elment inom ett element via w3Schools. Det funkade bra för det jag ville uppnå.
Jag kollade på andra sidor, andra exempel, på hur jag skulle kunna göra för att få upp 
en ny sida för användaren, men alla de som jag kollade på öppnade en helt ny sida åt användaren.
Vilket vi inte fick göra i den här uppgiften. :) 

Ett problem som uppstod för mig, var när jag skulle ta bort den inloggade sidan, eller den
sidan användaren blev skickad till om man skrev fel lösen. Jag använde mig först av att 
sätta style.display = "none", vilket tog bort elementen första gången. Andra gången jag
tryckte på logga in och sen ut, så var elementen fortfarande kvar. Jag såg då via utvecklar-
verktyget på google, att jag skapade nya element varje gång jag tryckte på knappen. 
Däför fick jag använda mig av removeChild - metoden, så att jag tog bort de element som jag
hade skapat. 

För att spara användaren så använde jag en variabel "inloggad" som sparades i localstorage,
som "true" eller "false", beroende på om användaren hade skrivit in rätt namn och lösen för
inlogg. Om användaren loggar ut så rensar man localstorage med clear().
Om inlogg = true, så kommer användaren mötas av att vara inloggad. Om inlogg inte är true,
så kommer användaren mötas av inlogg-sidan.


Jag tyckte uppgiften var rolig, men svår. Det tog en del tänkande. Jag hoppas att jag löste 
uppgiften på "rätt" sätt. Jag använde mig också av css på sidan, vilket jag tycker är väldigt
svårt att behärska. Mycket kod för lite design.. :)
